## 요약

<dl>
<dt>목표</dt>
<dd>사용자는 키보드를 사용하여 보다 쉽게 탐색할 수 있다.</dd>
<dt>할 일</dt>
<dd>반복되는 콘텐츠를 건너뛸 수 있는 수단을 제공하라.</dd>
<dt>중요성</dt>
<dd>키보드 인터페이스에 의존하는 사용자는 페이지를 효율적으로 이동할 수 있다.</dd>
</dl>


<Card title="성공 기준(SC)">
여러 [웹 페이지](../../../glossary/web-page)에 걸쳐 반복되는 콘텐츠 블록을 건너뛸 수 있는 [매커니즘](../../../glossary/mechanism)이 있다.
</Card>

### 의도

"이 성공 기준의 의도는 콘텐츠를 순차적으로 탐색하는 사람이 웹 페이지의 주요 콘텐츠에 곧바로 접근할 수 있도록 하는 것이다. 웹 페이지와 애플리케이션에는 종종 다른 페이지나 화면에 나타나는 콘텐츠가 있다. 반복되는 콘텐츠 블록의 예로는 탐색 링크, 헤더 콘텐츠 및 광고 프레임이 포함되지만 이에 국한하지 않는다. 개별 단어, 문구 또는 단일 링크와 같은 작은 반복 섹션은 이 조항의 목적상 블록으로 간주하지 않는다.

순차적으로 콘텐츠를 탐색하는 사용자는 일반적으로 각 페이지에서 반복되는 콘텐츠를 탐색해야 한다. 반대로 시력이 있는 사용자는 주요 콘텐츠가 일반적으로 나타나는 화면 중앙에 집중하여 반복되는 자료를 무시할 수 있다. 또한, 마우스 사용자는 한 번의 마우스 클릭으로 링크를 선택할 수 있다. 원하는 항목 앞에 오는 모든 링크 또는 양식 컨트롤을 만나야하는 사용자와는 대조적이다.

이 성공기준의 의도는 사용자 에이전트 기능과 중복되는 방법을 저작자가 제공하도록 요구하는 것은 아니다. 대부분의 웹 브라우저는 사용자 포커스를 페이지 상단으로 이동하는 키보드 단축키를 제공하므로 웹 페이지 하단에 탐색 링크 세트가 제공되는 경우 ""건너뛰기"" 링크를 제공하는 것은 불필요할 수 있다.

<Card title="참고">
이 성공 기준은 여러 페이지에서 반복되는 콘텐츠 블록을 다루지만, 성공 기준 1.3.1에 따라 개별 페이지의 구조적 마크업도 강력하게 장려한다.
</Card>

비록 성공 기준은 "웹 페이지 세트 내"라는 용어를 명확히 사용하지는 않지만, 집합에 속하는 페이지라는 개념은 암시되어 있다. 저작자는 어떤 식으로든 서로 관련이 없고 "공통의 목적을 공유하고 동일한 저작자, 그룹 또는 조직에 의해 만들어진 웹 페이지"(웹 페이지 세트의 정의)가 아닌 두 페이지에서 콘텐츠의 중복을 피할 것으로 기대하지 않는다.

<Card title="참고">
세트에 있지 않은 웹 페이지의 경우에도 페이지 내에서 반복되는 텍스트 블록이 있는 경우 이를 건너뛸 수 있는 수단을 제공하는 것이 도움이 될 수 있다(그러나 필수는 아님)."
</Card>

### 이점

이 성공 기준이 충족되지 않으면 일부 장애가 있는 사람이 웹 페이지의 주요 콘텐츠에 빠르고 쉽게 도달하는 것이 어려울 수 있다.

*   같은 사이트의 여러 페이지를 방문하는 스크린 리더 사용자는 주요 콘텐츠가 전달되기 전에 모든 페이지의 모든 헤더 내용과 수십 개의 탐색 링크를 듣지 않아도 된다.
*   키보드나 키보드 인터페이스만 사용하는 사람은 적은 키 입력으로 콘텐츠에 도달할 수 있다. 그렇지 않으면 메인 콘텐츠 영역의 링크에 도달하기 전에 수십 번의 키 입력을 해야 할 수 있다. 많은 시간이 걸릴 수 있으며 일부 사용자에게는 심각한 신체적 고통을 유발할 수 있다.
*   화면 확대기를 사용하는 사람은 새 페이지에 들어갈 때마다 주요 콘텐츠가 시작되는 곳을 찾기 위해 동일한 헤더 콘텐츠나 다른 정보 블록을 검색할 필요가 없다.
*   인지 장애가 있는 사람과 스크린 리더를 사용하는 사람은 링크가 목록으로 묶여 있을 때 도움이 될 수 있다.

### 예제

*   뉴스 조직의 홈페이지는 광고, 검색 및 기타 서비스를 위한 많은 블록과 사이드바로 둘러싸인 페이지 중앙에 메인 스토리가 포함되어 있다. 페이지 상단에 메인 스토리로 이동하는 링크가 있다. 이 링크를 사용하지 않으면 키보드 사용자는 메인 스토리에 도달하기 위해 약 40개의 링크를 탭해야 한다. 스크린 리더 사용자는 200개의 단어를 들어야 한다. 화면 확대기 사용자는 주요 콘텐츠의 위치를 검색해야 한다.
*   전자 상거래 웹 사이트에는 제품 검색 결과 이전에 긴 필터 목록이 포함되어 있다. 목록 위의 링크를 통해 사용자는 필터를 건너뛰고 제품 결과를 빠르게 확인할 수 있다.

### 기법

이 섹션의 각 번호가 매겨진 항목은 접근성 지침 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 판단하는 기법 또는 기법의 조합을 나타낸다. 기법은 기준의 최소 요구 사항을 넘어설 수 있다. 이러한 기법에서 포괄하지 않은 기준 충족의 다른 방법이 있을 수 있다. 다른 기법 사용에 대한 정보는 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참조하라.

#### 충분 기법

*   Creating links to skip blocks of repeated material using one of the following techniques:
    *   [G1: Adding a link at the top of each page that goes directly to the main content area](https://www.w3.org/WAI/WCAG22/Techniques/general/G1)
    *   [G123: Adding a link at the beginning of a block of repeated content to go to the end of the block](https://www.w3.org/WAI/WCAG22/Techniques/general/G123)
    *   [G124: Adding links at the top of the page to each area of the content](https://www.w3.org/WAI/WCAG22/Techniques/general/G124)
*   Grouping blocks of repeated material in a way that can be skipped, using one of the following techniques:
    *   [ARIA11: Using ARIA landmarks to identify regions of a page](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA11)
    *   [H69: Providing heading elements at the beginning of each section of content](https://www.w3.org/WAI/WCAG22/Techniques/html/H69)
    *   [PDF9: Providing headings by marking content with heading tags in PDF documents](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF9)
    *   [H64: Using the title attribute of the iframe element](https://www.w3.org/WAI/WCAG22/Techniques/html/H64)
    *   [SCR28: Using an expandable and collapsible menu to bypass block of content](https://www.w3.org/WAI/WCAG22/Techniques/client-side-script/SCR28)

#### 조언 기법

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

*   [C6: Positioning content based on structural markup](https://www.w3.org/WAI/WCAG22/Techniques/css/C6)
*   [H97: Grouping related links using the nav element](https://www.w3.org/WAI/WCAG22/Techniques/html/H97)
