<Card title="성공 기준(SC)">
[사용자 인터페이스 구성요소](../../../glossary/user-interface-component)가 초점을 받은 경우, [맥락의 변화](../../../glossary/changes-of-context)가 발생해서는 안 된다.
</Card>

## 요약

<dl>
<dt>목표</dt>
<dd>콘텐츠를 더욱 예측 가능하게 탐색할 수 있다.</dd>
<dt>할 일</dt>
<dd>항목에 초점이 있을 때 사용자의 맥락을 변경하지 않는다.</dd>
<dt>중요성</dt>
<dd>예측 가능하게 동작하는 콘텐츠는 장애인에게 특히 중요하다.</dd>
</dl>

### 의도

이 성공 기준의 목적은 방문자가 문서를 탐색할 때 기능을 예측할 수 있도록 하는 것이다. 초점을 받을 때 이벤트를 실행할 수 있는 모든 구성 요소는 맥락을 변경해서는 안 된다. 구성 요소가 초점을 받을 때 맥락을 변경하는 다음 사례를 포함하지만 이에 국한되지는 않는다.

*   구성 요소가 초점을 받으면 자동으로 제출되는 양식
*   구성 요소가 초점을 받으면 새 창이 열리는 경우
*   해당 구성 요소가 초점을 받으면 초점이 다른 구성 요소로 변경되는 경우

초점은 키보드(예: 컨트롤 탭 이동) 또는 마우스(예: 텍스트 필드 클릭)를 통해 컨트롤로 이동할 수 있다. 스크립팅에서 이 동작을 구현하지 않는 한 컨트롤 위로 마우스를 이동해도 포커스가 이동하지 않는다. 일부 컨트롤 유형의 경우 컨트롤을 클릭하면 컨트롤(예: 버튼)이 활성화되어 맥락 변화가 시작될 수도 있다.

<Card title="참고">
여기서 "구성요소"가 의미하는 것은 "사용자 인터페이스 요소" 또는 "사용자 인터페이스 구성요소"라고도 한다.
</Card>

### 이점

*   이 성공 기준은 상황의 변화가 예기치 않게 발생할 가능성을 줄여 시각 장애, 인지 장애, 운동 장애가 있는 사람들에게 도움이 된다.

### 예제

<dl>
    <dt>예 1: 드롭다운 메뉴</dt>
    <dd>
        페이지의 드롭다운 메뉴를 통해 사용자는 점프 대상 중에서 선택할 수 있다. 사용자가 키보드를 사용하여 선택 항목으로 이동하고 이를 활성화하면(스페이스바 또는 Enter 키 사용) 새 페이지로 이동한다.
        그러나 사용자가 선택 항목으로 이동하여 풀다운 메뉴에서 벗어나기 위해 Esc 키나 Tab 키를 누르는 경우 초점이 드롭다운 메뉴 밖으로 이동하므로 새 화면으로 이동하지 않는다.
    </dd>
    <dt>실패의 예: 도움말 대화 상자</dt>
    <dd>
        필드에 초점이 맞춰지면 해당 필드를 설명하고 옵션을 제공하는 도움말 대화 상자 창이 열린다. 키보드 사용자가 웹 페이지를 탭하면 대화 상자가 열리고 사용자가 필드를 탭하여 지나갈 때마다 키보드 초점이
        컨트롤에서 멀어진다.
    </dd>
</dl>

### 기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참고하라.

#### 충분 기법

*   [G107: Using "activate" rather than "focus" as a trigger for changes of context](https://www.w3.org/WAI/WCAG22/Techniques/general/G107)

<Card title="참고">
내용의 변화가 항상 [맥락의 변화](../../../glossary/changes-of-context)는 아니다. 콘텐츠의 변경이 맥락의 변경도 아닌 경우 이 성공 기준은 자동으로 충족된다.
</Card>

#### 조언 기법

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

*   [G200: Opening new windows and tabs from a link only when necessary](https://www.w3.org/WAI/WCAG22/Techniques/general/G200)
*   [G201: Giving users advanced warning when opening a new window](https://www.w3.org/WAI/WCAG22/Techniques/general/G201)
