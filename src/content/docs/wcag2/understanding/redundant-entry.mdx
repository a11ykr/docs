### 의도

이 성공 기준의 목적은 사용자가 여러 단계의 과정을 성공적으로 완료할 수 있도록 보장하는 것이다. 이는 과정 중에 정보가 한 번 이상 요구될 때 인지적 노력을 줄여준다. 또한 이전 단계에서 제공된 정보를 다시 떠올려야 할 필요성을 줄여준다.

입력을 위해 기억해야 하는 정보는 인지 또는 기억 장애가 있는 사용자에게 상당한 장벽이 될 수 있다. 모든 사용자는 여러 과정을 진행하면서 점진적인 정신적 피로를 자연스레 경험한다. 이러한 피로는 단기 작업 기억으로부터 정보를 회상해야 하는 스트레스로 인해 가속화된다. 학습 및 인지 장애가 있는 사용자는 정신적 피로에 매우 취약하다.

이전에 입력한 정보를 기억하도록 요구하면 사용자가 과정을 포기하게 하거나 동일한 정보를 잘못 재입력할 수 있다. 브라우저의 자동 완성 기능은 충분하지 않은데, 중복 입력에 대해 저장된 정보를 제공하거나 동일한 정보를 다시 요구하지 않아야 하는 것이 바로 콘텐츠(웹사이트)의 역할이기 때문이다.

이 성공 기준은 세션 간에 정보를 저장해야 한다는 요구 사항을 추가하지 않는다. [과정](../../../glossary/process)은 활동을 기반으로 정의되며, 사용자가 세션을 종료하거나 페이지를 벗어난 후 다시 돌아오는 경우에는 적용되지 않는다. 그러나 과정은 서로 다른 도메인에 걸쳐 실행될 수 있으므로, 결제 과정에 제3자 결제 제공업체가 포함된 경우 범위에 포함된다.

'선택 가능'이라는 용어는 규범적이지 않다. 이 용어는 자동 입력이 불가능한 경우 저작자가 개발할 수 있도록 허용한다. 이는 사용자에게 다음 작업을 수행할 수 있도록 허용하는 것이다.

- 드롭다운을 포함하여, 필드를 선택하고 정보를 입력하는 것.
- 페이지에서 텍스트를 선택하여 입력란에 복사하는 것.
- 체크박스를 선택하여 이전에 입력한 것과 동일한 값으로 입력란을 채우는 것(예: 청구 주소가 배송 주소와 동일함).

"선택 가능한" 데이터는 동일한 웹페이지에 있어야 한다. 이상적으로는 데이터가 요구되는 입력과 밀접하게 연관되어 기본적으로 표시되어야 한다. 하지만 표시/숨기기 컴포넌트 내부를 포함하여 페이지의 다른 곳에 있을 수도 있다.

이 성공 기준은 문서 형식의 이력서를 업로드하는 것과 같이 사용자가 다른 방식으로 데이터를 제공한 경우에는 적용되지 않는다.

이 성공 기준은 비밀번호 자동 입력을 허용하는 것이 충분 기법인 [접근 가능한 인증(최소)](../accessible-authentication-minimum.html)에 영향을 미치지 않는다. 이 경우 입력은 사용자의 브라우저에 의해 수행된다. '반복 입력'은 웹사이트 콘텐츠가 이전 입력 내용을 사용 가능할 수 있도록 요구하는 것이며, 세션 간의 저장이나 비밀번호 요구와 같은 필수적인 목적을 위한 것은 아니다.

이 성공 기준은 프라이버시나 개인 식별 정보(PII)에 특화된 요구 사항이나 예외 사항을 포함하지 않지만, 자동 입력과 같은 기법을 구현할 때 저작자는 과정 중에 정보를 일시적으로 저장하더라도 데이터 보호를 보장해야 한다. 추가적인 프라이버시 위험을 초래하지 않는 방식으로 중복 입력을 제거할 수 있으나, 이 성공 기준을 충족하기 위한 잘못된 구현으로 인해 추가적인 개인 식별 정보가 유출될 가능성도 있다.

다음의 경우는 예외이다

- 이전 답변이 제공되면 무효화되는 기억력 게임과 같이 정보를 다시 입력하는 것이 필수적인 경우
- 비밀번호 문자열이 표시되거나 복사되는 것을 방지하는 것과 같은 보안 조치, 비밀번호를 생성할 때 고유하고 복잡한 문자열이어야 하므로 저작자가 이를 검증할 수 없다. 시스템이 사용자에게 표시되지 않는 비밀번호를 수동으로 생성하도록 요구하는 경우, 사용자가 새로운 문자열을 다시 확인하도록 하는 것은 예외로 허용된다.
- 이전에 입력한 정보가 더 이상 유효하지 않은 경우, 사용자에게 해당 정보를 다시 입력하도록 요청할 수 있다.

### 이점

- 인지 장애가 있는 사용자는 단기 작업 기억의 어려움을 겪는다. 특정 정보를 반복해서 기억할 필요가 없으면 스트레스와 실수의 가능성이 줄어든다.
- 새로운 기억을 형성하거나 정보를 회상하는 데 어려움을 겪는 사용자 및 인지와 관련된 다른 기능에 제약이 있는 사용자가 자신의 기억에 불필요하게 의존하지 않고도 과정을  완료할 수 있다.
- 스위치 제어 또는 음성 입력 등을 사용하는 지체 장애가 있는 사용자는 텍스트 입력의 필요성이 줄어드는 이점을 얻는다.

### 예제

- 새 컴퓨터를 구매하는 여러 단계의 과정 중 첫 번째 단계에서 사번(ID)을 요청한다. 세 번째 단계에서 사용자는 컴퓨터가(동료가 아닌) 본인의 소유인지 확인하도록 요청받으며, 사번이 다시 표시된다. 사용자가 사번을 변경할 수 있도록 허용하되, 기본값은 이전에 입력한 번호로 설정된다.
- 전자상거래 웹사이트의 양식에서 사용자가 청구 주소와 배송 주소가 동일한지 확인할 수 있다.
- 사용자가 잘못된 신용카드 번호가 입력된 결제 양식을 제출한다. 페이지가 갱신되면서 오류 메시지가 표시된다. 신용카드 번호와 같이 이미 제출된 정보는 양식에서 삭제되지 않는다.
- 검색 결과 페이지는 동일한 과정 내에서 이전에 입력한 검색어로 검색 입력란을 미리 채운다.

### 기법

이 섹션의 각 번호가 매겨진 항목은 접근성 지침 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 판단하는 기법 또는 기법의 조합을 나타낸다. 기법은 기준의 최소 요구 사항을 넘어설 수 있다. 이러한 기법에서 포괄하지 않은 기준 충족의 다른 방법이 있을 수 있다. 다른 기법 사용에 대한 정보는 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참조하라.
