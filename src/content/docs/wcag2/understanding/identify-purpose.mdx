### 의도

이 성공 기준의 목적은 페이지에 있는 많은 요소의 목적이 프로그래밍 방식으로 판별될 수 있도록 보장하여 사용자 에이전트가 다양한 양식을 사용하여 해당 목적을 추출하고 사용자에게 제시할 수 있도록 하는 것이다.

제한된 어휘를 가진 많은 사용자는 웹을 사용하기 위해 익숙한 용어나 기호에 의존한다. 그러나 한 사용자에게 익숙한 것이 다른 사용자에게는 익숙하지 않을 수도 있다. 저작자가 목적을 표시하면 사용자는 개인화 및 사용자 기본 설정을 활용하여 자신에게 친숙한 기호 또는 어휘 세트를 로드할 수 있다.

이 성공 기준에서는 저작자가 아이콘, 영역 및 구성 요소(예: 버튼, 링크, 필드)의 목적을 프로그래밍 방식으로 연결하여 사용자 에이전트가 각 목적을 결정하고 표시기 또는 용어를 사용자가 이해할 수 있도록 조정할 수 있도록 해야 한다. 이는 이 맥락을 제공하는 의미 체계 또는 메타데이터를 추가하여 달성된다. 역할 정보를 추가하는 것과 유사하지만(4.1.2에서 요구됨) UI 구성요소가 무엇인지에 대한 정보(예: 이미지)를 제공하는 대신 구성요소가 나타내는 내용에 대한 정보(예: 홈페이지 링크)를 제공한다.

페이지의 영역을 식별하면 사람들이 사용자 에이전트를 사용하여 영역을 제거하거나 강조 표시할 수 있다.

목소리가 없는 사람들을 위한 제품은 사용자의 의사소통을 돕기 위해 기호를 사용하는 경우가 많다. 이 기호는 사실 사람들의 언어이다. 불행하게도 이러한 기호 중 다수는 저작권의 적용을 받으며 상호 운용이 불가능하다. 즉, 최종 사용자는 하나의 장치만 사용할 수 있으며 단일 회사에서 만들지 않은 콘텐츠, 앱 또는 보조 기술을 사용할 수 없다.

이 적합 기준은 기호 사용자가 한 회사에서 만든 것이 아닌 다양한 콘텐츠를 이해할 수 있도록 기호의 상호 운용을 가능하게 한다. 사용자 기호가 동일한 노드에 매핑되면 사용자 에이전트는 사용자가 이해할 수 있는 기호를 로드할 수 있다. 그런 다음 사람들은 기호를 구입하여 다양한 장치나 응용 프로그램에서 사용할 수 있다. (기호는 여전히 독점적이지만 상호 운용이 가능하다는 점에 유의하라.)

### 이점

다음과 같은 다양한 인지 장애가 있는 사람들에게 도움이 된다.

*   기억
*   집중력과 주의력
*   언어 관련
*   집행 기능 및 의사 결정

이 성공 기준을 충족하면 다음과 같은 추가 지원이나 친숙한 인터페이스가 필요한 사용자에게 도움이 된다.

*   사용자에게 친숙한 기호 및 그래픽
*   기능이 적고 인지 과부하가 적음
*   키보드 단축키

### 예제

*   웹사이트는 [ARIA 랜드마크](https://www.w3.org/WAI/ARIA/apg/practices/landmark-regions/)를 사용해 페이지의 영역을 식별하고, 사용자는 `main` 역할이 없는 영역을 숨길 수 있다.
*   웹 사이트 탐색의 링크는 사용자가 자신의 아이콘을 추가할 수 있도록 표시되어 있다.
*   웹 사이트의 아이콘은 사용자가 페이지에 자신의 아이콘 세트로 대체할 수 있도록 표시된다.

### 기법

이 섹션의 각 번호가 매겨진 항목은 접근성 가이드라인 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 판단하는 기법 또는 기법의 조합을 나타낸다. 기법은 기준의 최소 요구 사항을 넘어설 수 있다. 이러한 기법에서 포괄하지 않은 기준 충족의 다른 방법이 있을 수 있다. 다른 기법 사용에 대한 정보는 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참조하라.

#### 충분 기법

*   Programmatically indicating the purpose of icons, regions and user interface components
*   [ARIA11: Using ARIA landmarks to identify regions of a page](https://www.w3.org/WAI/WCAG22/Techniques/aria/ARIA11) AND
*   Using microdata to markup user interface components (future link)

#### 조언 기법

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

*   Enabling user agents to find the version of the content that best fits their needs
*   Using semantics to identify important features (e.g., `coga-simplification="simplest"`)
*   Using `aria-invalid` and `aria-required`
