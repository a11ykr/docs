### 의도

이 성공 기준의 목적은 사용자 에이전트가 의미를 이해하는 데 필요한 읽기 순서를 유지하면서 콘텐츠의 대체 표현을 제공할 수 있도록 하는 것이다. 의미가 있는 콘텐츠의 순서를 하나 이상 프로그래밍 방식으로 판별하는 것이 가능하도록 만드는 것이 중요하다. 이 성공 기준을 충족하지 않는 콘텐츠는 보조 기술이 잘못된 순서로 콘텐츠를 읽거나 대체 스타일 시트 또는 기타 형식 변경이 적용될 때 사용자에게 혼란을 주거나 헤매게 만들 수 있다.

의미에 영향을 주지 않고 내용의 순서를 변경할 수 없다면 순서는 *의미*가 있다. 예를 들어 페이지에 두 개의 독립적인 기사가 포함되어 있는 경우 기사가 서로 엮여서 제시되지 않는다면 기사의 상대적 순서는 의미에 영향을 미치지 않을 수 있다. 이러한 상황에서는 기사 자체에는 의미 있는 순서가 있을 수 있지만 기사를 담는 용기에는 의미 있는 순서가 없을 수 있다.

일부 요소의 의미는 해당 콘텐츠가 의미 있는 순서인지를 정의한다. 예를 들어 HTML에서 텍스트는 항상 의미 있는 순서이다. 테이블(table)과 순서가 있는 목록(ol)은 의미 있는 순서이지만 순서가 없는 목록(ul)은 그렇지 않다.

콘텐츠 정렬 순서가 항상 의미 있는 것은 아니다. 예를 들어, 웹 페이지의 기본 섹션과 탐색 섹션의 상대적 순서는 의미에 영향을 주지 않는다. 프로그래밍 방식으로 판별된 읽기 순서에서 어느 순서로든 발생할 수 있다. 또 다른 예로, 잡지 기사에는 여러 개의 부가 정보 영역(callout sidebars)이 포함되어 있다. 기사와 부가 정보 영역의 순서는 의미에 영향을 주지 않는다. 이러한 경우 성공 기준을 충족할 수 있는 웹 페이지 읽기 순서는 다양하다.

명확하게:

*   특정 선형 순서를 제공하는 것은 의미에 영향을 미치는 경우에만 필요하다.
*   "올바른" 순서가 두 개 이상 있을 수 있다(WCAG 2.0 정의에 따르면).
*   올바른 순서는 하나만 제공하면 된다.

### 이점

*   이 성공 기준은 콘텐츠를 소리내어 읽는 보조 기술에 의존하는 사람들에게 도움이 될 수 있다. 기본 표현의 정보 순서에서 분명한 의미는 내용이 음성 형식으로 표현될 때에도 동일할 것이다.

### 예제

*   **예제 1:** 다단 문서에서 콘텐츠의 선형 표현은 단(열) 상단에서 하단으로 흐른 후 다음 단의 상단으로 이어진다.
*   **예제 2:** CSS는 탐색 모음(navigation bar), 페이지의 주요 요소 및 부가 요소를 배치하는 데 사용된다. 섹션의 시각적 표현은 프로그래밍 방식으로 판별된 순서와 일치하지 않지만 페이지의 의미는 섹션의 순서에 따라 달라지지 않는다.

### 기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참고하라.

#### 충분 기법

*   [G57: Ordering the content in a meaningful sequence](https://www.w3.org/WAI/WCAG22/Techniques/general/G57) for all the content in the Web page
*   Marking sequences in the content as meaningful using one of the following techniques **AND** [G57: Ordering the content in a meaningful sequence](https://www.w3.org/WAI/WCAG22/Techniques/general/G57) for those sequences
    *   [H34: Using a Unicode right-to-left mark (RLM) or left-to-right mark (LRM) to mix text direction inline](https://www.w3.org/WAI/WCAG22/Techniques/html/H34)
    *   [H56: Using the dir attribute on an inline element to resolve problems with nested directional runs](https://www.w3.org/WAI/WCAG22/Techniques/html/H56)
    *   [C6: Positioning content based on structural markup](https://www.w3.org/WAI/WCAG22/Techniques/css/C6)
    *   [C8: Using CSS letter-spacing to control spacing within a word](https://www.w3.org/WAI/WCAG22/Techniques/css/C8)
*   [C27: Making the DOM order match the visual order](https://www.w3.org/WAI/WCAG22/Techniques/css/C27)
*   [PDF3: Ensuring correct tab and reading order in PDF documents](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF3)
