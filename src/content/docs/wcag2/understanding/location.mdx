## [성공기준 이해 2.4.8](https://w3.org/TR/WCAG22#location):위치 제공 

(Level AAA)

<Card title="성공 기준(SC)"></Card>

웹 사이트가 [웹 페이지 세트](../../../glossary/set-of-web-pages)로 구성되어 있는 경우, 사용자 위치 정보를 제공해야 한다.

## 요약

<dl>
<dt>목표</dt>
<dd>사용자가 자신이 일련의 페이지 중 어디에 있는지 알고 있다.</dd>
<dt>할 일</dt>
<dd>탐색 경로(breadcrumbs), 사이트 맵 또는 기타 지표를 사용하여 위치 정보를 제공하라.</dd>
<dt>중요성</dt>
<dd>위치 표시기는 인지 장애가 있는 사람들의 혼란을 줄여준다.</dd>
</dl>

### 의도

이 성공 기준의 목적은 사용자가 일련의 웹 페이지, 웹 사이트 또는 웹 애플리케이션 내에서 방향을 잡고 관련 정보를 찾을 수 있는 방법을 제공하는 것이다.

### 이점

*   이 성공 기준은 웹 페이지에 대한 일련의 긴 탐색 단계를 따를 때 혼란스러울 수 있는 짧은 집중 시간을 가진 사람들에게 유용하다. 또한 사용자가 링크를 따라 웹 페이지 집합 내의 깊은 페이지로 직접 이동하고 해당 페이지의 콘텐츠를 이해하거나 더 많은 관련 정보를 찾기 위해 해당 웹 사이트를 탐색해야 하는 경우에도 유용하다.

### 예제

사이트에서 사용자가 자신의 위치를 ​​확인하는 데 도움이 되는 링크

연구 그룹은 대학 교육 부서의 일부이다. 그룹 홈페이지는 학과 홈페이지, 대학 홈페이지로 연결된다.

탐색 경로(breadcrumb trail)

포털 웹 사이트는 주제별 범주로 구성된다. 사용자가 카테고리와 하위 카테고리를 탐색할 때 탐색경로는 카테고리 계층 구조의 현재 위치를 보여준다. 각 페이지에는 포털 홈 페이지에 대한 링크도 포함되어 있다.

### 관련 자료

### 기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참고하라.
