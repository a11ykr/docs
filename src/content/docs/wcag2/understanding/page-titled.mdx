<Card title="성공 기준(SC)">
[웹 페이지](../../../glossary/web-page)는 주제나 목적을 설명하는 제목을 포함해야 한다.
</Card>


### 의도
이 성공 기준의 의도는 각 웹페이지의 의미가 들어있는 제목을 제공하여, 사용자가 콘텐츠를 찾고 그 안에서 위치를 파악할 수 있도록 돕는 것이다. 제목은 사용자가 페이지 콘텐츠를 읽거나 해석하지 않아도 현재 위치를 식별할 수 있게 한다. 제목이 사이트 맵이나 검색 결과 목록에 나타날 때, 사용자는 필요한 콘텐츠를 더 빠르게 식별할 수 있다. 사용자 에이전트는 사용자가 페이지를 식별할 수 있도록 페이지 제목을 쉽게 제공한다. 예를 들어, 사용자 에이전트는 창의 제목 표시줄이나 페이지가 포함된 탭의 이름으로 페이지 제목을 표시할 수 있다.

페이지가 문서나 웹 애플리케이션인 경우, 그 제목만으로도 페이지의 목적을 설명하기에 충분하다. 문서나 웹 애플리케이션의 이름을 반드시 제목으로 사용할 필요는 없다. 다른 요소가 페이지의 목적이나 주제를 설명할 수도 있다.

단일 페이지 애플리케이션(SPA)과 같이 다양한 별도의 페이지/뷰가 동일한 URI에서 제공되고 페이지 콘텐츠가 동적으로 변경되는 경우, 페이지 제목 또한 현재 뷰의 콘텐츠나 주제를 반영하여 동적으로 변경되어야 한다.

[성공 기준 2.4.4](link-purpose-in-context)와 [2.4.9](link-purpose-link-only)는 링크 목적을 다루며, 그 중 다수는 웹페이지로 연결되는 링크이다. 여기서도 연결되는 문서나 웹 애플리케이션의 제목이 링크 목적을 설명하기에 충분하다. 링크와 제목이 일치하거나 매우 유사한 것은 좋은 관례이며, '클릭된' 링크와 사용자가 도착한 웹페이지 간의 연속성을 제공한다

### 이점

* 이 기준은 모든 사용자가 웹페이지에 포함된 정보가 자신의 요구와 관련이 있는지 빠르고 쉽게 식별할 수 있게 한다.
* 시각 장애가 있는 사람들은 여러 웹페이지가 열려 있을 때 콘텐츠를 구별할 수 있어 도움이 된다.
* 인지 장애, 단기 기억력 제한, 읽기 장애가 있는 사람들도 제목으로 콘텐츠를 식별할 수 있어 도움이 된다.
* 또한 웹페이지간 이동 시 오디오에 의존하여 동작하는 중증 운동 장애인에게도 도움이 된다.

        *   문서모음의 주요 섹션은 "X 지침 이해하기" 및 "X 성공 기준 이해하기"라는 제목의 페이지이다.
        *   부록 A의 제목은 "용어집"이다.
        *   부록 B의 제목은 "감사의 말"이다.
        *   부록 C의 제목은 "참고자료"이다.
    </dd>
    <dt>웹 애플리케이션</dt>
    <dd>
        뱅킹 애플리케이션을 통해 사용자는 은행 계좌를 확인하고, 과거 명세서를 조회하고, 거래를 수행할 수 있다. 웹 애플리케이션은 각 웹페이지의 제목을 동적으로 생성한다. 예: "XYZ 은행, Alex Smith 계좌", "XYZ 은행, 계좌 1234-5678의 2005년 12월 명세서"
    </dd>

### 기법

이 섹션의 각 번호가 매겨진 항목은 접근성 지침 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 판단하는 기법 또는 기법의 조합을 나타낸다. 기법은 기준의 최소 요구 사항을 넘어설 수 있다. 이러한 기법에서 포괄하지 않은 기준 충족의 다른 방법이 있을 수 있다. 다른 기법 사용에 대한 정보는 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참조하라.

#### 충분 기법

*   [G88: Providing descriptive titles for Web pages](https://www.w3.org/WAI/WCAG22/Techniques/general/G88) **그리고** 다음 기술 중 하나를 사용하여 제목을 웹 페이지와 연결한다.
    *   [H25: Providing a title using the title element](https://www.w3.org/WAI/WCAG22/Techniques/html/H25)
    *   [PDF18: Specifying the document title using the Title entry in the document information dictionary of a PDF document](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF18)

#### 조언 기법

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

*   [G127: Identifying a Web page's relationship to a larger collection of Web pages](https://www.w3.org/WAI/WCAG22/Techniques/general/G127)
