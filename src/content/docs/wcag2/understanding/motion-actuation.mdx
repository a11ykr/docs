## 요약

<dd>콘텐츠는 사용자가 장치를 움직이는 능력에 의존하지 않는다.</dd>
<dt>할 일</dt>
<dd>페이지의 콘텐츠를 제어하는데 장치 움직임에만 의존하지 않아야 한다.</dd>
<dt>중요성</dt>
<dd>어떤 사람들은 장치를 안정적으로 잡거나 움직일 수 없다.</dd>

장치나 사용자의 움직임으로 실행할 수 있는 [기능](../../../glossary/functionality)은 [사용자 인터페이스 구성요소](../../../glossary/user-interface-component)로도 작동할 수 있어야 하고, 실수로 작동하는 것을 방지하기 위해서 움직임에 반응하는 것을 비활성화 시킬 수 있어야 한다.

### 의도

이 성공 기준의 목적은 장치를 움직이거나(예: 흔들기, 기울이기), 또는 장치를 향한 제스처를 통해(카메라와 같은 센서가 제스처를 감지하고 해석하는 경우) 실행되는 기능을 보다 일반적인 사용자 인터페이스 구성 요소로도 조작할 수 있도록 보장하는 것이다.

<Card title="참고">
이 기준은 장치를 향한 몸짓, 기울이기, 흔들기와 같이 동작에 직접 반응하는 센서를 통한 입력과 관련된다.
이는 (지리적) 위치 센서나 비콘에 의해 기록되는 사용자의 공간 이동이나, 사용자의 의도적인 제스처가 아닌, 장치에서 관찰되는 이벤트에는 적용되지 않는다.
또한 키보드, 포인터 또는 보조 기술을 조작하는 과정에서 발생하는 부수적인 움직임도 포함하지 않는다.
</Card>

장치에는 휴대전화나 태블릿의 가속도계 및 자이로스코프와 같이 입력으로 활용될 수 있는 센서가 포함되어 있는 경우가 많다.
이러한 센서를 사용하면 사용자는 장치의 방향을 바꾸거나 특정 방식으로 움직이는 것만으로도 기능을 제어할 수 있다.
다른 경우에는 웹 콘텐츠가 카메라나 기타 센서를 통해 사용자의 제스처를 해석하여 기능을 실행할 수도 있다.
예를 들어 장치를 흔들어 “실행 취소” 명령을 실행하거나, 부드럽게 손을 넘겨서 일련의 페이지 내에서 앞뒤로 이동할 수 있다.
일부 장애가 있는 사용자는 장치가 고정된 거치대(예: 휠체어)에 장착되어 있거나 운동 장애로 인해 이러한 센서를 전혀 또는 충분히 정밀하게 조작할 수 없다.
따라서 움직임을 통해 제공되는 기능은 다른 매커니즘에서도 사용할 수 있어야 한다.

또한 떨림이나 기타 운동 장애로 인해 의도치 않게 센서를 활성화시키는 사용자도 있다.
사용자는 이러한 우발적인 기능 실행을 방지하기 위해 움직임 기반 실행을 끌 수 있어야 한다.
응용 프로그램은 사용자가 시스템 수준에서 동작 감지를 비활성화할 수 있도록 하는 운영 체제 설정을 지원함으로써 이 요구 사항을 충족할 수 있다.

움직임이 기능에 필수적이거나, 움직임이나 제스처를 사용하지 않으면 과업이 무의미해지는 경우에는 예외를 적용한다.
일부 애플리케이션은 장치 센서 데이터를 사용하도록 특별히 제작된다.
예를 들어 장치의 움직임을 사용하여 걸음 수를 계산하는 만보계는 이 요구 사항에서 제외된다.

### 이점

*   이 성공 기준은 장치가 장착되어 있거나 사용자가 필요한 신체 움직임을 수행할 수 없어 특정 동작(기울이기, 흔들기, 제스처 등)을 수행하지 못하는 사람들에게 도움이 된다. 또한 이 성공 기준은 사용자가 터치나 보조 기술과 같은 다른 수단을 통해 계속 모든 기능을 조작할 수 있도록 보장한다.
*   다른 사용자 역시 장치를 움직일 수 없는 상황에서 이점을 누릴 수 있다.

### 예제

*   사용자는 애플리케이션 설정으로 흔들어 실행 취소와 움직임 기반 기능을 끌 수 있다.
*   텍스트를 필드에 입력한 후 장치를 흔들면 입력을 취소할 수 있는 대화 상자가 표시된다. 텍스트 필드 옆의 취소 버튼도 동일한 기능을 제공한다.
*   사용자는 장치를 기울여 다음 페이지나 이전 페이지로 이동할 수 있다. 동일한 기능을 수행하는 버튼도 제공된다.
*   사용자는 장치를 움직이거나(move) 이동시켜(pan) 상호작용 가능한 사진의 보기를 변경할 수 있다. 동일한 기능을 수행하는 컨트롤도 제공한다.
*   사용자는 콘텐츠를 탐색하기 위해 장치를 향해 제스처를 취할 수 있다. 컨트롤을 통해서도 탐색할 수 있다.

### 기법

이 섹션의 각 번호가 매겨진 항목은 접근성 지침 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 판단하는 기법 또는 기법의 조합을 나타낸다. 기법은 기준의 최소 요구 사항을 넘어설 수 있다. 이러한 기법에서 포괄하지 않은 기준 충족의 다른 방법이 있을 수 있다. 다른 기법 사용에 대한 정보는 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참조하라.

#### 충분 기법

*   [G213: Provide conventional controls and an application setting for motion activated input](https://www.w3.org/WAI/WCAG22/Techniques/general/G213)
*   GXXX: Supporting system level features which allow the user to disable motion actuation

#### 오류

다음은 접근성 지침 실무 그룹에서 이 성공 기준의 실패로 간주하는 일반적인 실수이다.

*   [F106: Failure due to inability to deactivate motion actuation](https://www.w3.org/WAI/WCAG22/Techniques/failures/F106)
*   FXXX: Failure of Success Criterion 2.5.4 due to disrupting or disabling system level features which allow the user to disable motion actuation
