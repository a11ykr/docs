이 섹션에서는 WCAG 2 성공 기준을 충족하기 위해 사용할 수 있는 메타데이터 기술에 대해 설명한다. 메타데이터에 대한 자세한 내용은 아래 리소스를 참조하라.

가장 기본적인 수준에서 메타데이터는 본질적으로 '데이터에 관한 데이터'이며 리소스를 설명하고 찾는 데 사용된다.

메타데이터는 웹 페이지와 웹 페이지의 액세스 가능한 구성 요소를 설명하고 웹 콘텐츠의 대체 버전을 서로 연결하는 데 사용할 수 있는 강력한 도구이다. 이러한 설명을 통해 사용자는 필요하거나 선호하는 특정 정보를 찾을 수 있다.

WCAG 2와 함께 메타데이터는 다음과 같은 다양한 역할을 수행할 수 있다.

1.  메타데이터는 준수하는 웹 페이지의 대체 버전을 준수하지 않는 웹 페이지와 연결하는 데 사용될 수 있다. 이를 통해 사용자는 사용할 수 없는 미준수 페이지에 접속할 때 준수하는 대체 버전을 찾을 수 있다.
2.  메타데이터는 개발된 페이지의 여러 버전이 있는 경우, 특히 대체 페이지가 다양한 장애가 있는 개인을 위해 최적화된 경우 대체 페이지를 찾고 설명하는 데 사용될 수 있다. 사용자는 메타데이터를 사용하여 대체 버전을 찾고 버전의 특성을 식별함으로써 자신의 요구 사항에 가장 적합한 버전을 찾을 수 있다.
3.  메타데이터는 위의 #1 및 #2와 같이 전체 페이지에 사용되는 것 외에도 페이지 하위 구성 요소의 대체 버전을 설명하는 데 사용될 수 있다. 다시 말하지만, 메타데이터는 웹 페이지 하위 구성 요소의 대체 버전을 찾는 데 사용될 수 있을 뿐만 아니라 사용자의 요구 사항에 가장 적합한 버전을 판별하기 위해 대체 버전(여러 개가 있는 경우)에 대한 설명을 얻는 데에도 사용할 수 있다.

## 메타데이터 리소스

메타데이터 설명은 종종 리소스의 주제나 출판 날짜와 같이 정의되고 합의된 어휘의 값을 제공하며 기계 판독이 가능하다. 사용 중인 메타데이터 체계를 이해하는 소프트웨어는 다른 방법으로는 실현할 수 없는 유용한 작업을 수행할 수 있다. 전형적으로, 메타데이터를 갖는 객체는 그러한 메타데이터 설명을 하나 이상 가질 수 있다.

메타데이터에 대해 잘 알려진 사양(스키마)은 다음과 같다.

*   [Metadata Encoding and Transmission Standard (METS) scheme](http://www.loc.gov/standards/mets/)
*   [Dublin Core Metadata Initiative (DCMI) terms for cross-disciplinary resources](http://dublincore.org)
*   [IEEE Standards](http://www.ieee.org/index.html)


리소스 설명을 제공하는 데 사용할 수 있는 몇 가지 도구가 있거나 수동으로 제공할 수 있다. 리소스 생성 시점이나 출판 시점에 메타데이터를 더 쉽게 생성하고 수집할 수 있을수록 프로세스가 더 효율적이고 발생 가능성이 높아진다.

Some examples include:

*   [DC-dot](http://www.ukoln.ac.uk/metadata/dcdot/)

Accessibility metadata implementations include:

*   [The Inclusive Learning Exchange (TILE)](http://barrierfree.ca/tile/)
