<Card title="성공 기준(SC)">
각 [웹 페이지](../../glossary/web-page)의 기본 [인간 언어](../../glossary/human-language)는 [프로그래밍 방식으로 판별](../../glossary/programmatically-determined)될 수 있어야 한다.

## 요약

<dl>
<dt>목표</dt>
<dd>보조 기술은 페이지의 언어를 판별할 수 있다.</dd>
<dt>할 일</dt>
<dd>페이지에서 주로 사용되는 언어를 표시한다.</dd>
<dt>중요성</dt>
<dd>보조 기술을 사용하는 사람들은 올바른 언어로 정보를 얻는다.</dd>
</dl>

### 의도

이 성공 기준의 목적은 콘텐츠 개발자가 사용자 에이전트가 텍스트 및 기타 언어 콘텐츠를 올바르게 표시하는 데 필요한 정보를 웹 페이지에 제공하도록 하는 것이다. 보조 기술과 기존 사용자 에이전트 모두 웹 페이지의 언어가 식별되면 텍스트를 더 정확하게 렌더링할 수 있다. 화면 낭독기는 올바른 발음 규칙을 불러올 수 있다. 시각적 브라우저는 문자와 스크립트를 올바르게 표시할 수 있다. 미디어 플레이어는 자막을 올바르게 표시할 수 있다. 결과적으로 장애가 있는 사용자가 콘텐츠를 더 잘 이해할 수 있게 된다.

웹 페이지의 기본 인간 언어는 [국제화 모범 사례: XHTML 및 HTML 콘텐츠에서 언어 지정(Internationalization Best Practices: Specifying Language in XHTML & HTML Content)](https://www.w3.org/International/techniques/authoring-html#gslang)에 설명된 기본 텍스트 처리 언어이다. 웹 페이지가 여러 언어를 사용하는 경우 기본 텍스트 처리 언어는 가장 많이 사용되는 언어이다. (여러 언어를 동일하게 사용하는 경우 처음 사용하는 언어를 기본 인간 언어로 선택해야 한다.)

<Card title="참고">
레벨 A 준수를 목표로 하는 다국어 사이트의 경우 실무 그룹은 개발자가 성공 기준 3.1.2를 따르도록 강력히 권장한다. 성공 기준 3.1.2가 수준 AA 성공 기준이더라도 마찬가지이다.
</Card>

### 이점

이 성공 기준은 다음의 경우에 도움이 된다.

*   화면 낭독기나 텍스트를 합성 음성으로 변환하는 기타 기술을 사용하는 사람
*   문자와 알파벳을 인식하거나 단어를 해독하는 등 서면(written) 자료를 유창하고 정확하게 읽는 데 어려움을 겪는 사람
*   텍스트 음성 변환 소프트웨어를 사용하는 특정 인지, 언어 및 학습 장애가 있는 사람
*   동기화된 미디어의 자막에 의존하는 사람

### 예제

<dl>
    <dt>예 1. 두 가지 언어로 된 콘텐츠가 포함된 웹 페이지</dt>

    </dd>

### 기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참고하라.

#### 충분 기법

*   [H57: Using the language attribute on the HTML element](https://www.w3.org/WAI/WCAG22/Techniques/html/H57)
*   [PDF16: Setting the default language using the /Lang entry in the document catalog of a PDF document](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF16)
*   [PDF19: Specifying the language for a passage or phrase with the Lang entry in PDF documents](https://www.w3.org/WAI/WCAG22/Techniques/pdf/PDF19)

#### 조언 기법

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

*   [SVR5: Specifying the default language in the HTTP header](https://www.w3.org/WAI/WCAG22/Techniques/server-side-script/SVR5)
