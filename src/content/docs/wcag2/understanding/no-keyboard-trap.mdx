(Level A)

<Card title="성공 기준(SC)">
[키보드 인터페이스](../../../glossary/keyboard-interface)를 사용하여 키보드 초점을 페이지의 구성요소로 이동할 수 있는 경우, 키보드 인터페이스만으로도 해당 구성요소에서 초점을 이동시킬 수 있어야 한다. 수정되지 않은 화살표, 탭 키, 또는 다른 표준 종료 방법이 필요한 경우, 사용자에게 초점을 이동시키는 방법에 대해 안내해야 한다.

    <Card title="참고">
    이 성공기준을 충족하지 못하는 콘텐츠는 모든 페이지를 사용하는 사용자들의 능력을 방해할 수 있다. 웹 페이지의 모든 콘텐츠는 다른 성공기준의 충족 여부와 관계없이 반드시 이 성공기준을 준수해야 한다. [준수 요구사항 5: 불간섭](https://www.w3.org/TR/WCAG22/#cc5)을 참고하라.
    </Card>
</Card>

## 요약

<dl>
<dt>목표</dt>
<dd>키보드 사용자는 막히지 않는다.</dd>
<dt>할 일</dt>
<dd>사용자가 구성 요소에서 벗어나 탐색하는 방법을 항상 알 수 있도록 하라.</dd>
<dt>중요성</dt>
<dd>키보드에 의존하는 사람들은 탐색할 다른 수단이 없는 경우가 많다.</dd>
</dl>

### 의도

이 성공기준의 목적은 콘텐츠가 웹 페이지의 콘텐츠 하위 섹션에 키보드 초점을 "가두는" 일이 없도록 하는 것이다. 이는 여러 형식이 결합된 페이지에서 플러그인이나 임베디드 애플리케이션을 사용하여 렌더링할 때 흔히 발생하는 문제이다.

웹 페이지의 기능상 콘텐츠의 하위 섹션으로 초점을 제한하는 경우가 있을 수 있지만, 사용자가 그 상태에서 벗어나도록 초점을 "풀어낼" 수 있는 방법을 알고 있다면 허용된다.

### 이점

*   시각 장애인과 지체 장애가 있는 사람을 포함하여 키보드나 키보드 인터페이스를 사용하여 웹을 사용하는 사람들.

### 예제

<dl>
    <dt>달력 위젯</dt>
    <dd>
        달력 위젯을 사용하면 사용자는 키보드를 사용하여 달력에 항목을 추가, 제거 또는 업데이트할 수 있다. 위젯의 컨트롤은 웹 페이지 내 탭 순서의 일부이므로 사용자는 위젯의 컨트롤뿐만 아니라 이후에 나오는 모든 링크나 컨트롤을 탭으로 이동할 수 있다.
    </dd>
    <dt>퍼즐 애플릿</dt>
    <dd>
        사용자가 애플릿을 탭하면 추가 탭 및 기타 키 입력이 애플릿에 의해 처리된다. 애플릿을 종료하는 데 사용되는 키 입력을 설명하는 지침은 애플릿 자체 내에서뿐만 아니라 애플릿 전에서 제공된다.
    </dd>
    <dt>모달 대화 상자</dt>
    <dd>
        웹 응용 프로그램이 대화 상자를 표시한다. 대화 상자 하단에는 취소와 확인이라는 두 개의 버튼이 있다. 대화 상자가 열리면 초점이 대화 상자 내에 갇힌다. 대화 상자의 마지막 컨트롤에서 탭을 누르면 대화 상자의 첫 번째 컨트롤에 초점이 맞춰진다. 취소 버튼이나 확인 버튼을 활성화하면 대화 상자가 닫힌다.
    </dd>
</dl>

### 기법

이 섹션의 각 번호가 매겨진 항목은 접근성 지침 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 판단하는 기법 또는 기법의 조합을 나타낸다. 기법은 기준의 최소 요구 사항을 넘어설 수 있다. 이러한 기법에서 포괄하지 않은 기준 충족의 다른 방법이 있을 수 있다. 다른 기법 사용에 대한 정보는 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참조하라.

#### 충분 기법

*   [G21: Ensuring that users are not trapped in content](https://www.w3.org/WAI/WCAG22/Techniques/general/G21)

다음은 접근성 지침 실무 그룹에서 이 성공 기준의 실패로 간주하는 일반적인 실수이다.
