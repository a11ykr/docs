다음 표는 HTML 문서에서 ARIA 마크업 사용에 대한 문서 준수 요구사항을 정의하고, HTML 요소에 대한 암묵적 ARIA 의미론을 식별한다. 또한, 이러한 요소에 대한 암묵적 ARIA 의미론은 [HTML AAM](#bib-html-aam-1.0 "HTML Accessibility API Mappings 1.0")에서 정의한다.

각 언어 기능(요소)은 첫 번째 열의 셀에서 두 번째 열의 셀에 주어진 ARIA 의미론(역할, 상태 및 속성)을 암시합니다. 각 행의 세 번째 셀은 저작자가 요소에 지정할 수 있는 ARIA `role` 값과 `aria-*` 속성을 정의합니다. 세 번째 열의 셀에 **Any**`role`이라는 용어가 포함된 경우, 저작자가 요소에 사용할 수 있는 모든 `role` 값이 _MAY_ 지정될 수 있음을 나타냅니다. 그러나 저작자가 요소의 내재적 역할, `generic` 역할, 또는 ARIA에 의해 비권장된 역할을 지정하는 것은 _권장하지 않음_입니다. 세 번째 열의 셀에 **No `role`**이라는 용어가 포함된 경우, 저작자가 내재적 ARIA 의미론이나 HTML 요소의 고유 의미론을 덮어쓰는 것은 _MUST NOT_입니다.

<div class="addition">
	WAI-ARIA 1.2")는 저작자가 지정할 수 없는 금지된 상태 및 속성을 가진 역할을 식별합니다. 이러한 역할은 저작자가 특정 WAI-ARIA 속성을 지정하는 것을 허용하지 않습니다. 이러한 내재적 WAI-ARIA 역할을 노출하는 HTML 요소도 저작자가 이러한 WAI-ARIA 속성을 지정하는 것을 허용하지 않습니다.

	저작자가 명명할 수 없는 요소는 `aria-label`혹은 `aria-labelledby`속성을 지정할 수 없는 요소이다. 이러한 요소는 암묵적 역할을 저작자가 명시적인 WAI-ARIA 역할로 덮어쓰지 않는 한, 명명할 수 없다. 자세한 정보는 4.1 ARIA 속성을 사용하여 요소 명명참고.
</div>

<Aside title="참고">
	암묵적 ARIA 의미와 일치하는 ARIA `role`및 `aria-*`속성을 설정하는 것은 _권장하지 않는다_. 그러나 일부 상황에서는 이러한 속성을 명시적으로 설정하는 것이 도움이 될 수 있다. 예를 들어, 특정 요소에 대한 암묵적 ARIA 의미론을 노출하지 않는 사용자 에이전트의 경우가 있다.
</Aside>

<Aside title="참고">
다음 표에 정의된 Digital Publishing(Dpub) WAI-ARIA Module 1.0 `role` 값을 사용하는 것은 적합하지만, 이러한 역할을 웹 사이트 구현 목적으로 사용하는 것은 의도와 맞지 않는다. 디지털 출판 산업 이외의 목적으로 이러한 역할을 사용하는 경우에는 사용자에게 의도된 경험을 제공하는지 확인하기 위해 추가적인 테스트가 필요하다.
</Aside>

These features can be used to make accessibility tools render content to their users in more useful ways. For example, ASCII art, which is really an image, appears to be text, and in the absence of appropriate roles and properties would end up being rendered by screen readers as a nonsensical string of punctuation characters. Using the features described in this section, one can instead make the ATs skip the ASCII art and just read the caption:

```html
<figure>
	<pre role="img" aria-label="ASCII Fish" >
  o           .'`/
    '      /  (
  O    .-'` ` `'-._      .')
      _/ (o)        '.  .' /
      )       )))     ><  <
      `\  |_\      _.'  '. \
        '-._  _ .-'       '.)
    jgs     `\__\
	</pre>
	<figcaption id="fish-caption">Joan G. Stark, "<cite>fish</cite>" . October 1997. ASCII on electrons. 28×8. </figcaption>
</figure>
```
