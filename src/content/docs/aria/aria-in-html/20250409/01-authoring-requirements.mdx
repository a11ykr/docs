## 1. HTML에서 ARIA 사용에 대한 저작자 요구사항

HTML에서 ARIA를 사용하는 저작자는 다음 규칙을 준수해야 한다:

1.  **ARIA `role` 또는 `aria-*` 속성을 사용하지 마라**, 만약 해당 의미 체계나 동작을 제공하는 네이티브 HTML 요소나 속성을 사용할 수 있다면.
2.  **요소의 암시적 네이티브 의미 체계를 변경하지 마라.**
    예를 들어, HTML `<h1>` 요소에 `role=button`을 부여하지 마라.
    <Aside type="note">
    참고: 요소의 암시적 네이티브 의미 체계를 변경하는 것은 ARIA의 첫 번째 규칙을 위반하는 것이다.
    </Aside>
3.  **모든 대화형 ARIA 컨트롤은 키보드로 접근 가능해야 한다.**
    만약 요소가 네이티브로 포커스 가능하지 않다면 (예: `div`, `span`, `p`), 요소에 `tabindex="0"`을 추가하여 포커스 순서에 포함시켜야 한다. (더 자세한 정보는 ARIA의 다섯 번째 규칙을 참조하라.)
4.  **`role="presentation"` 또는 `aria-hidden="true"`를 포커스 가능한 요소에 사용하지 마라.**
    포커스 가능한 요소에 `aria-hidden="true"`를 사용하면 일부 사용자가 포커스를 받은 요소와 상호작용하지 못하게 될 수 있다.
    포커스 가능한 요소에 `role="presentation"`을 사용하면 요소의 의미 체계가 제거되어 일부 사용자가 요소의 목적을 이해하지 못하게 될 수 있다.
5.  **모든 대화형 요소에는 접근 가능한 이름이 있어야 한다.**
    예를 들어, `button` 요소나 `aria-role="button"`이 있는 요소는 접근 가능한 이름이 있어야 한다.
