사용자가 고유 기능에 대한 단일 컨트롤로 인식하는 콘텐츠의 일부

<Aside title="참고 1">
다중 사용자 인터페이스 구성요소는 단일 프로그래밍 요소로 구현될 수 있다. 여기에서 구성요소는 프로그래밍 기법이 아니라 사용자가 별도의 컨트롤로 인식하는 것을 말한다.
</Aside>
<Aside title="참고 2">
사용자 인터페이스 구성요소에는 스크립트로 생성한 구성요소뿐만 아니라 서식(<span lang="en">form</span>) 요소 및 링크도 포함된다.
</Aside>
<Aside title="참고 3">
여기에서 “구성요소” 또는 “사용자 인터페이스 구성요소”가 의미하는 것은 때때로 “사용자 인터페이스 요소(<span lang="en">user interface element</span>)”라고도 한다.
</Aside>

<Card title="예제" icon="open-book">
애플릿(<span lang="en">applet</span>)은 선이나 페이지 또는 무작위로 콘텐츠를 이동하는 데 사용될 수 있는 “컨트롤”를 가지고 있다. 이들 각각은 별도의 이름이 있어야 하고 독립적으로 설정할 필요가 있기 때문에, 각각 “사용자 인터페이스 구성요소”가 될 것이다.
</Card>
