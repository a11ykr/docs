사용자가 인식하지 못한 채 실행되면 전체 페이지를 동시에 볼 수 없는 사용자에게 혼란을 줄 수 있는 중요한 변화

맥락에서의 변화는 다음과 같은 변경을 포함한다.

1.  [사용자 에이전트](../user-agents "사용자를 위해 웹 콘텐츠를 검색하고 표시하는 소프트웨어")
1.  [뷰포트](../viewport "사용자 에이전트가 콘텐츠를 표시하는 객체")
1.  초점(focus)
1.  [웹 페이지](../web-page-s "HTTP를 사용하여 단일 URI에서 얻어진 내장 안 된 리소스(non-embedded resource)과 렌더링에 사용되거나 사용자 에이전트에서 렌더링되도록 의도된 모든 다른 리소스")의 의미를 변경하는 콘텐츠

<Aside title="참고">
콘텐츠가 변경되었다고 해서 항상 맥락이 변경된 것은 아니다. 확장 개요(<span lang="en">outline</span>), 동적 메뉴 또는 탭 제어와 같은 콘텐츠의 변경은 앞에서 언급한 것 중 하나(예: 초점)가 변경되지 않는 한 반드시 맥락이 변경된 것은 아니다.
</Aside>

<Card title="예제" icon="open-book">
새 창 열기, 다른 구성요소로 초점 이동, 새 페이지로 이동(사용자가 새 페이지로 이동한 것처럼 보이는 모든 항목 포함) 또는 페이지 내용을 대대적으로 재배치하는 것 등이 맥락 변경의 예이다.
</Card>
