<dt>할 일</dt>

<dd>페이지 제목은 사용자가 다양한 페이지를 식별하고 구별하는 데 도움이 된다.</dd>

### 의도

이 성공 기준의 목적은 각 웹 페이지에 설명적인 제목이 있는지 확인하여 사용자가 콘텐츠를 찾고 그 안에서 방향을 잡을 수 있도록 돕는 것이다. 제목은 사용자가 페이지 콘텐츠를 읽거나 해석하지 않고도 현재 위치를 식별한다. 사이트 맵이나 검색 결과 목록에 제목이 나타나면 사용자는 필요한 콘텐츠를 더 빨리 식별할 수 있다. 사용자 에이전트는 사용자가 페이지를 식별할 수 있도록 페이지 제목을 쉽게 사용할 수 있도록 한다. 예를 들어, 사용자 에이전트는 창 제목 표시줄에 페이지 제목을 표시하거나 페이지를 포함하는 탭 이름으로 표시할 수 있다.

페이지가 문서 또는 웹 애플리케이션인 경우 문서 또는 웹 애플리케이션의 이름은 페이지의 목적을 설명하기에 충분하다. 문서나 웹 애플리케이션의 이름을 반드시 사용할 필요는 없다. 다른 것들도 페이지의 목적이나 주제를 설명할 수 있다.

SPA(단일 페이지 애플리케이션, Single Page Applications)와 같이 다양한 개별 페이지/뷰가 모두 동일한 URI에서 명목상 제공되고 페이지 콘텐츠가 동적으로 변경되는 경우 페이지 제목도 콘텐츠 또는 현재 보이는 주제를 반영하도록 동적으로 변경되어야 한다.

[성공 기준 2.4.4](link-purpose-in-context) 및 [2.4.9](link-purpose-link-only)는 링크의 목적을 다루고 있으며 그 중 대부분은 웹 페이지에 대한 링크이다. 여기서도 링크되는 문서나 웹 애플리케이션의 이름만으로도 링크의 목적을 충분히 설명할 수 있다. 링크와 제목이 일치하거나 매우 유사한 것은 좋은 습관이며 '클릭한' 링크와 사용자가 방문하는 웹 페이지 사이의 연속성을 제공한다.

### 이점

*   이 기준은 사용자가 웹 페이지에 포함된 정보가 자신의 요구 사항과 관련이 있는지 여부를 빠르고 쉽게 식별할 수 있다는 점에서 모든 사용자에게 도움이 된다.
*   시각 장애가 있는 사용자는 여러 웹 페이지가 열려 있을 때 콘텐츠를 구별할 수 있다는 이점을 누릴 수 있다.
*   인지 장애, 제한된 단기 기억 및 읽기 장애가 있는 사람들도 제목으로 콘텐츠를 식별하는 기능의 이점을 누릴 수 있다.
*   이 기준은 또한 웹 페이지 간을 탐색할 때 오디오에 의존하여 작동하는 심각한 이동 장애가 있는 사람들에게도 도움이 된다.

### 예제

HTML 웹 페이지

HTML 웹 페이지의 설명 제목은 &lt;title&gt; 요소로 표시되어 사용자 에이전트의 제목 표시줄에 표시된다.

문서 컬렉션

[WCAG 2.1 이해](./)의 제목은 "WCAG 2.1 이해"이다.

*   소개 페이지에는 "WCAG 2.0 이해 소개"라는 제목이 있다.
*   문서의 주요 섹션은 "지침 X 이해" 및 "성공 기준 X 이해"라는 제목의 페이지이다.
*   부록 A에는 "용어집"이라는 제목이 있다.
*   부록 B에는 "감사의 말씀"이라는 제목이 있다.
*   부록 C에는 "참고자료"라는 제목이 있다.

웹 애플리케이션

뱅킹 애플리케이션을 사용하면 사용자는 은행 계좌를 확인하고, 과거 명세서를 보고, 거래를 수행할 수 있다. 웹 애플리케이션은 각 웹 페이지에 대한 제목을 동적으로 생성한다(예: "XYZ 은행, Alex Smith 계좌" "XYZ 은행, 2005년 12월 계좌 1234-5678 명세서").

### 기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 [WCAG 성공 기준에 대한 기법 이해](understanding-techniques), 특히 "기타 기법" 섹션을 참고하라.

#### 충분 기법

*   [G88: Providing descriptive titles for Web pages](https://www.w3.org/WAI/WCAG22/Techniques/general/G88) **그리고** 다음 기술 중 하나를 사용하여 제목을 웹 페이지와 연결한다.
    
    *   [H25: Providing a title using the title element](https://www.w3.org/WAI/WCAG22/Techniques/html/H25)

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

*   [G127: Identifying a Web page's relationship to a larger collection of Web pages](https://www.w3.org/WAI/WCAG22/Techniques/general/G127)
