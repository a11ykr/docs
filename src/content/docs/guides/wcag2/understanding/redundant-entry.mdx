## [성공기준 이해 3.3.7](https://w3.org/TR/WCAG22#redundant-entry):중복 입력 

(Level A)

<Card title="성공 기준(SC)"></Card>

사용자가 이전에 입력했거나 사용자에게 제공된 정보 중 동일한 [과정](../../../../glossary/process)으로 다시 입력해야 하는 정보는 다음 중 하나를 준수해야 한다.

*   자동 완성
*   사용자 선택 가능

다음의 경우는 예외로 한다.

*   정보를 재입력하는 것이 [필수적인](../../../../glossary/essential) 경우
*   콘텐츠의 보안을 보장하기 위해 해당 정보가 필요한 경우
*   이전에 입력한 정보가 더 이상 유효하지 않을 경우

## 요약

<dl>
<dt>목표</dt>
<dd>사용자가 여러 단계의 절차를 더 쉽게 완료할 수 있도록 한다.</dd>
<dt>할 일</dt>
<dd>동일한 세션에서 동일한 정보를 두 번 요청하지 않는다.</dd>
<dt>중요성</dt>
<dd>인지 장애가 있는 일부 사람들은 이전에 입력한 내용을 기억하는 데 어려움을 겪는다.</dd>
</dl>

### 의도

이 성공 기준의 목적은 사용자가 여러 단계의 절차를 성공적으로 완료할 수 있도록 보장하는 것이다. 절차 중에 두 번 이상 요청되는 정보에 대한 인지적 노력이 줄어듭니다. 또한 이전 단계에서 제공된 정보를 기억할 필요성도 줄어든다.

입력을 위해 기억해야 하는 정보는 인지 또는 기억 장애가 있는 사용자에게 심각한 장벽이 될 수 있다. 모든 사용자는 절차의 단계를 진행하면서 자연스럽게 점진적인 정신적 피로를 경험한다. 이러한 피로는 단기 작업 기억에서 정보를 회상하는 스트레스로 인해 가속화한다. 학습 및 인지 장애가 있는 사용자는 정신적 피로에 매우 취약하다.

사람들에게 이전에 입력한 정보를 기억하도록 요구하면 동일한 정보를 포기하거나 부정확하게 다시 입력하게 될 수 있다. 브라우저의 자동완성 기능만으로는 충분하지 않은데, 이는 중복 입력에 대해 저장된 정보를 제공하거나 동일한 정보를 요구하지 않도록 하는 것이 콘텐츠(웹사이트) 측의 역할이기 때문이다.

이 성공 기준은 세션 간에 정보를 저장하기 위한 요구 사항을 추가하지 않는다. A [과정](../../../../glossary/process)은 활동을 기반으로 정의되며 사용자가 세션을 닫거나 다른 곳으로 이동한 후 돌아올 때는 적용되지 않는다. 그러나 절차는 여러 도메인에 걸쳐 실행될 수 있으므로 결제 과정에 제3자 결제 제공업체가 포함된 경우 해당 범위에 속한다.

"선택 가능"이라는 용어는 규범적이지 않다. 이 용어를 사용하면 저작자는 자동 채우기가 불가능한 기술을 개발할 수 있다. 여기에는 사용자에게 다음을 허용하는 것이 포함될 수 있다.

*   드롭다운을 포함하여 필드를 선택하고 채운다.
*   페이지에서 텍스트를 선택하고 입력에 복사한다.
*   이전에 입력한 것과 동일한 값으로 입력을 채우려면 체크박스를 선택한다(예: 청구서 수신 주소가 배송 주소와 동일함).

"선택할 수 있는" 데이터는 동일한 페이지에 있어야 한다. 이상적으로는 기본적으로 표시되고 데이터가 필요한 입력과 밀접하게 연관된다. 그러나 표시/숨기기 구성 요소를 포함하여 페이지의 다른 곳에 있을 수도 있다.

이력서를 문서 형식으로 업로드하는 등 사용자가 다른 방법으로 데이터를 제공하는 경우에는 이 성공 기준이 적용되지 않는다.

이 성공 기준은 비밀번호 자동 완성을 허용하는 것만으로도 충분한 기술인 [접근 가능한 인증(최소)](accessible-authentication-minimum.html)에 영향을 미치지 않는다. 이 경우 채우기는 사용자의 브라우저에 의해 수행된다. 중복 항목은 이전 항목을 사용할 수 있도록 웹 사이트 콘텐츠를 요청하지만 세션 사이나 비밀번호 요청과 같은 필수 목적을 위한 것은 아니다.

이 기준에는 개인 정보 보호 또는 개인 식별 정보(PII: personally identifiable information)와 관련된 요구 사항이나 예외가 포함되어 있지 않지만, 자동 채우기와 같은 기술을 구현할 때 저작자는 해당 과정 중 정보를 일시적으로 저장할 때 데이터 보호를 보장해야 한다. 추가 개인 정보 보호 위험을 초래하지 않는 방식으로 중복 항목을 제거하는 것이 가능하지만, (이 기준을 충족하기 위한) 잘못된 구현으로 인해 추가 PII가 유출될 수도 있다.

다음과 같은 예외가 있다.

*   이전 답변이 제공되면 무효화되는 기억력 게임과 같은 항목에 대한 재입력은 필수적인 사용이다.
*   비밀번호 문자열이 표시되거나 복사되는 것을 방지하는 등의 보안 조치. 비밀번호를 생성할 때 비밀번호는 고유하고 복잡한 문자열이어야 하므로 저작자가 확인할 수 없다. 시스템에서 사용자에게 표시되지 않는 비밀번호를 수동으로 생성하도록 요구하는 경우 사용자가 새 문자열을 다시 검증하도록 하는 것은 예외적으로 허용된다.
*   이전에 입력한 정보가 더 이상 유효하지 않은 경우, 해당 정보를 다시 입력하도록 사용자에게 요청할 수 있다.

### 이점

*   인지 장애가 있는 사용자는 단기 작업 기억 장애를 겪는다. 특정 정보를 반복적으로 기억할 필요가 없으므로 스트레스와 실수 가능성이 줄어든다.
*   새로운 기억 형성, 정보 회상, 인지와 관련된 기타 기능에 어려움을 겪는 사용자는 불필요하게 기억에 의존하지 않고도 절차를 완료할 수 있다.
*   스위치 제어나 음성 입력 등 이동 장애가 있는 사용자는 텍스트 입력 필요성이 줄어드는 이점을 누릴 수 있다.

### 예제

*   새 컴퓨터를 구입하기 위한 과정 중 첫 번째 단계에서 사용자의 기업 식별 번호(ID)를 요청하는 양식이 있다. 3단계에서는 컴퓨터가 동료가 아닌 사용자의 소유인지 확인하라는 요청을 받고 ID를 다시 표시한다. 사용자가 ID를 변경할 수 있지만 기본값은 이전에 입력한 ID이다.
*   전자상거래 웹사이트의 양식을 통해 사용자는 청구서 수신 주소와 배송 주소가 동일한 주소인지 확인할 수 있다.
*   검색 결과 페이지는 동일한 과정에서 이전에 입력한 검색어로 검색 입력을 미리 채운다.
