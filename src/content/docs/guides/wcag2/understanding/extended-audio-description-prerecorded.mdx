[오디오 해설](../../../../glossary/audio-description)이 비디오의 느낌을 전달하는 데 전경 오디오의 일시정지(pause)만으로는 충분하지 않은 경우, [동기화된 미디어](../../../../glossary/synchronized-media)에 포함된 모든 [사전 녹화된](../../../../glossary/prerecorded) [비디오](../../../../glossary/video) 콘텐츠에 대해 [확장형 오디오 해설](../../../../glossary/extended-audio-description)을 제공해야 한다.
</Card>

## 요약

<dl>
<dt>목표</dt>
<dd>보다 자세한 오디오 해설과 함께 비디오를 재생할 수 있다.</dd>
<dt>할 일</dt>
<dd>비디오의 시각적 콘텐츠에 대해 확장된 오디오 해설을 제공한다.</dd>
<dt>중요성</dt>
<dd>시각 장애가 있거나 시각적 내용을 이해하지 못하는 사람들은 비디오를 재생하면서 설명을 들을 수 있다.</dd>
</dl>

### 의도

이 성공 기준의 목적은 시각 장애가 있는 사람들에게 표준 오디오 해설로 제공할 수 있는 것 이상으로 동기화된 미디어에 대한 접근성을 제공하는 것이다. 이는 동기화된 미디어를 주기적으로 정지하고 추가 오디오 해설을 재생함으로써 수행된다. 그 후 동기화된 미디어가 재개된다.

추가적인 설명이 필요하지 않은 사람의 시청을 방해하기 때문에 기능을 켜고 끌 수 있는 기술이 제공되는 경우가 많다. 또는 추가 설명이 있는 버전과 없는 버전이 제공될 수 있다.

### 이점

*   전맹, 화면을 볼 수 없는 저시력, 현재 일어나고 있는 일을 시각적으로 해석하는 데 어려운 인지 장애를 가진 사람들은 종종 시각적 정보에 대한 오디오 설명을 사용한다. 그러나 대화가 너무 많으면 오디오 해설이 충분하지 않다. 확장된 오디오 해설은 비디오를 이해하는 데 필요한 추가 정보를 제공할 수 있다.

### 예제

*   *예시 1. 강의 영상.* 물리학 교수가 강의를 하고 있다. 교수는 화이트보드에 손으로 스케치를 하고, 그림을 그리면서 빠르게 말한다. 한 문제에 대한 토론이 끝나면 교수는 그림을 지우고 말과 몸짓을 계속하면서 또 다른 스케치를 그린다. 문제 사이에는 비디오가 일시 중지되고 교수의 그림과 동작에 대한 자세한 오디오 설명이 제공된다. 그런 다음 비디오가 재개된다.

### 관련 자료

자료는 정보 제공 목적으로만 제공되며 보증을 암시하지 않는다.

*   [Synchronized Multimedia Integration Language (SMIL) 1.0](https://www.w3.org/TR/REC-smil/)
*   [Synchronized Multimedia Integration Language (SMIL) 2.0](https://www.w3.org/TR/SMIL/)
*   [Accessibility Features of SMIL](https://www.w3.org/TR/SMIL-access/)
*   [NCAM Rich Media Accessibility, Accessible SMIL Templates](http://ncam.wgbh.org/invent_build/web_multimedia/accessible-digital-media-guide/guideline-h-multimedia#techH12)
*   [Standard Techniques in Audio Description](http://joeclark.org/access/description/ad-principles.html)

### 기법

이 섹션에서 번호가 매겨진 각 항목은 WCAG 실무 그룹이 이 성공 기준을 충족하기에 충분하다고 간주하는 기법 또는 기법의 조합을 나타낸다. 그러나 이러한 특정 기법을 사용할 필요는 없다. 다른 기법 사용에 대한 자세한 내용은 [WCAG 성공 기준에 대한 기법 이해](../understanding-techniques), 특히 "기타 기법" 섹션을 참고하라.

#### 충분 기법

*   [G8: Providing a movie with extended audio descriptions](https://www.w3.org/WAI/WCAG22/Techniques/general/G8) using one of the following:
    *   [SM1: Adding extended audio description in SMIL 1.0](https://www.w3.org/WAI/WCAG22/Techniques/smil/SM1)
    *   [SM2: Adding extended audio description in SMIL 2.0](https://www.w3.org/WAI/WCAG22/Techniques/smil/SM2)
    *   Using any player that supports audio and video

#### 조언 기법

준수를 위해 필수는 아니지만 콘텐츠에 더 쉽게 접근할 수 있도록 다음과 같은 추가 기법을 고려해야 한다. 모든 기법을 사용할 수 없거나 모든 상황에서 효과적인 것은 아니다.

*   [H96: Using the track element to provide audio descriptions](https://www.w3.org/WAI/WCAG22/Techniques/html/H96)
